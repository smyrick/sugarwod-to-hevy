Architecture: sugarwod-to-hevy

Last updated: 2026-06-18 — Structural overview for developers and AI agents. detail: small theme: neutral light/dark

Overview

A single-file Python 3 CLI that converts a SugarWOD workout export (workouts.csv, one row per workout) into the Strong app CSV format (one row per set) that Hevy accepts via Import Strong CSV. No third-party dependencies — standard library only.

The hard part is not parsing; it is weight-unit correctness. Hevy's Strong importer assumes the Weight column is kilograms, while SugarWOD logs barbell loads in pounds. The converter writes adaptive-precision kg strings that round-trip back to the original whole-pound display in Hevy (see lbs_to_strong_kg()).

Core Functions

Everything lives in one module. Grouped by responsibility:

FunctionResponsibility
main / convert_fileCLI parsing, read input, orchestrate, write 12-column Strong CSV, print summary counts
convert_rowTop-level router: score_type == "Load" → load path, else WOD path
convert_load_rowBarbell lifts → one row per set with weight + reps
convert_wod_rowTimed / rep / generic WODs → custom-exercise rows (Seconds or Reps)

format_strong_weightDispatch on input/output unit combo (lbs↔kg)
dedupe_rowsCollapse duplicate (date,title) rows, keeping the richest set_details

build_workout_notes / build_wod_notesCompose pipe-separated note strings (notes, RX/SCALED, PR, result)
EXERCISE_NAME_MAPStatic dict mapping SugarWOD lift names → Hevy canonical names (e.g. Back Squat → Squat (Barbell))

Migration pitfalls (weight units, Hevy quirks, rounding) are captured in docs/LEARNINGS.md. Personal workout CSVs are gitignored (input/, output/, and *.csv).

Weight Conversion (why it exists)

The Strong schema has a single Weight column and no unit field. Hevy reads it as kilograms. Writing raw pounds makes loads appear ~2.2× too heavy. lbs_to_strong_kg() rounds to the nearest whole pound, then picks the shortest kg string (2–3 decimals in practice; algorithm tries up to 5) whose round-trip kg → lbs redisplays the original pound value in Hevy.

SugarWOD lbsCSV kgHevy display
320145.15320
19086.183190
335151.953335

Verified: every whole pound 1–1000 round-trips exactly, needing at most 3 decimals.

Testing

tests/test_convert.py (run with python3 -m pytest) covers the correctness-critical paths: the lbs_to_strong_kg round-trip for every whole pound 1–1000, format_strong_weight unit dispatch, build_load_sets pairing branches, score_type routing, dedupe, and the end-to-end "bad rows skip, not crash" behavior. pytest is a dev-only dependency; the converter itself stays stdlib-only. Hevy-verified session spot-checks remain documented in docs/LEARNINGS.md.
